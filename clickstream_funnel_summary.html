Clickstream Funnel
System Design

Our goal is to identify user clicks that successfully convert into actual sales, and map the exact engineering steps required to do it.

At a scale of 100M+ daily events, stateless, high-volume click tracking cannot simply be continuously updated against stateful, low-volume purchase ledgers without triggering massive write congestion and data warehouse performance degradation.

The Blueprint
This design details a decoupled, Medallion Lakehouse Architecture on GCP engineered to isolate streaming ingress, resolve cross-device identities, and materialize end-to-end conversion paths cleanly and cost-effectively.

Storage Strategy

Bifurcated Storage - Cold & Hot Paths

Splitting arriving tracking items to balance low cost retention with immediate query access.

  • ▹
    The Cold Path: Groups telemetry files every five minutes into Cloud Storage blocks using compressed Apache Parquet formats.
  • ▹
    The Hot Path: Directs streaming messages into BigQuery staging environments via the continuous Storage Write API.
  • ▹
    Cost Management: Columnar compression on the cold path cuts resource footprints by over eighty percent.
Architect's Note
Cold path parquet objects serve as our immutable ledger, allowing for full state reconstruction if needed.

Identity Stitching Resolution

Connecting anonymous guest sessions to authenticated user profiles through a non destructive relational bridge.

Pipeline Logic

The Transformation Split Logic

How our hourly scheduled SQL pipeline parses the raw staging JSON and routes data into decoupled warehouse entities.

1. Loading fact_clickstream (Processes All Rows)
SELECT session_id, event_type, 
       -- Downstream JSON path extraction expressions
       LAX_STRING(raw_payload.payload.product_id) AS product_id
FROM stg_clickstream_events;
2. Loading fact_orders (Isolates Success Ledger Rows Alone)
SELECT LAX_STRING(raw_payload.payload.order_id) AS order_id,
       session_id, 
       LAX_NUMERIC(raw_payload.payload.gross_amount) AS gross_amount
FROM stg_clickstream_events
WHERE event_type = 'purchase_success';
Filtering specifically for purchase parameters safeguards transaction tables from web traffic noise, using the session identifier as an immutable star schema link.

Pipeline Schedule

Hourly Orchestration Engine

How our data pipelines execute automated dependencies securely every sixty minutes without data corruption.

  • ▹
    Lightweight Conductor: Managed Apache Airflow coordinates dependencies as a DAG, pushing heavy computation parameters to our serverless warehouse data loops.
